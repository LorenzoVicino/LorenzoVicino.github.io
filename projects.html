This is the project I want to represent my portfolio right now: a GTFS multi-city ticketing platform that joins transport data, itinerary logic, and digital ticketing in one system.

Main Project

  • GTFS Multicity Ticketing

    PostgreSQL, Next.js, Docker, GTFS, OpenStreetMap, digital ticketing

    According to the repository README, the project starts from a relational PostgreSQL persistence model for public transport timetables, itinerary management with changes, and digital ticket sales and validation.

    The current repository also includes a full-stack Next.js web app with city search, GTFS loading from database, map rendering of stops and routes, direct GTFS zip upload, and automatic ticket catalog generation for imported agencies.

    • multi-city GTFS data model with transport entities scoped by city
    • itinerary creation with one or two segments and transfer handling
    • ticket purchase and validation flows, including signed QR validation
    • departures, bookings, itineraries, and ticket detail APIs
    • bootstrap flow with Dockerized PostgreSQL and bundled GTFS import

Why It Matters

I like this project because it sits at the intersection of data modeling, backend design, APIs, operational workflows, and a usable interface for real transport information.
